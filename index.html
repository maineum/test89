Hello World!

The background of this app contains a few large files and may take a moment to show up.

This might be a little more complex than some "Hello World" applications but Blend4Web, by its nature, utilizes HTML, CSS, Javascript and the content authored in Blender. With the advent of WebGL, what was once a web page can now be more like a program we are used to running on our computer. This page loads the Blend4Web application into a background DIV element that is stationary so it does not scroll. A short CSS file tells your browser what to do with the background DIV element as well as the scrollable text area you see now. This is just normal HTML text that can be edited in the HTML file. It is inside a DIV element that is positioned overlapping the Blend4Web app.

Once your scene is created in Blender, you export it using the JSON method. The HTML file calls up the Javascript file which loads the JSON information into the background DIV element using its identifier; "background_app". The CSS file also uses this ID to target the DIV element. The Blend4Web app is controlled using the Javascript file. Since this is a very simple app, the JS file is short. This scene has no camera controls. The camera animation is done in Blender. I should also mention that you can export a fully self contained HTML file from Blender using Blend4Web. No Javascrip or CSS files would be required. That is ideal in some cases but has limitations in how much control you have over how it loads and less flexibilty than controling your app with code.

This project has been kept simple, however many things could be done to improve how it scales and runs on various devices. On important detail is the image dmensions you use with WebGL should be POT (power of two numbers like 2, 4, 8, 16, 32 etc). The image demension of the background texture in this app are 2048 X 4098.

One of the ideas behind this project is to create a page that will load even if the WebGL part can not load for some reason. If the connection is slow, the text will load and the background app will show up when it can.

You can use the files for this app as a template to start your own project. If you create your own Blender scene and export it in this application's directory with the JSON method, the page would load your scene into the background. Make sure you overwrite the JSON and bin files with th same names.
